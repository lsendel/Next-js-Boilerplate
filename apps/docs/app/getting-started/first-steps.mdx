## Try the Authentication System

The boilerplate includes a fully functional authentication system:

### Test the Sign-In Flow

1. Click **"Sign In"** in the navigation
2. Default provider is Clerk (keyless mode for development)
3. Create a test account or sign in with:
   - Email/Password
   - Social OAuth (Google, GitHub, etc.)
   - Magic links

### Switch Authentication Providers

You can switch between auth providers **without changing code**:

```bash
# Edit apps/web/.env.local
NEXT_PUBLIC_AUTH_PROVIDER=clerk      # Default
# NEXT_PUBLIC_AUTH_PROVIDER=cloudflare # Cloudflare Access
# NEXT_PUBLIC_AUTH_PROVIDER=cognito    # AWS Cognito
# NEXT_PUBLIC_AUTH_PROVIDER=test       # E2E testing only
```

## Database Exploration

The app uses **PGlite** for local development (no Docker required!).

### Open Drizzle Studio

```bash
pnpm --filter web db:studio
```

This opens Drizzle Studio at [https://local.drizzle.studio](https://local.drizzle.studio)

### View Database Tables

You'll see these tables:
- `users` - User accounts
- `sessions` - Active sessions
- `passwordResetTokens` - Password recovery
- `userPreferences` - User settings
- `tenants` - Multi-tenancy support
- `counter` - Demo table


## View Error Monitoring

The app includes Sentry Spotlight for development error monitoring:

### Start Spotlight

```bash
pnpm --filter web dev:spotlight
```

Access at [http://localhost:3000/monitoring](http://localhost:3000/monitoring)

### Trigger a Test Error

Add this to any component:

```tsx
<button onClick={() => {
  throw new Error('Test error for Sentry')
}}>
  Trigger Error
</button>
```

The error appears in Spotlight instantly!

## Database Migrations

When you modify the database schema:

### 1. Edit the Schema

Open `apps/web/src/server/db/models/Schema.ts`:

```typescript
export const myNewTable = pgTable('my_new_table', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  createdAt: timestamp('created_at').defaultNow(),
})
```

### 2. Generate Migration

```bash
pnpm --filter web db:generate
```

This creates a new migration file in `apps/web/migrations/`

### 3. Apply Migration

```bash
pnpm --filter web db:migrate
```

Or restart the dev server (auto-migrates on start).

## Environment Variables

The app uses **T3 Env** for type-safe environment variables.

### View All Variables

Check `apps/web/src/libs/Env.ts` for all available variables.

### Add Your Own

```typescript
// In apps/web/src/libs/Env.ts
export const Env = createEnv({
  server: {
    MY_API_KEY: z.string().min(1),
  },
  client: {
    NEXT_PUBLIC_MY_PUBLIC_KEY: z.string().min(1),
  },
  runtimeEnv: {
    MY_API_KEY: process.env.MY_API_KEY,
    NEXT_PUBLIC_MY_PUBLIC_KEY: process.env.NEXT_PUBLIC_MY_PUBLIC_KEY,
  },
})
```

Then add to `apps/web/.env.local`:

```bash
MY_API_KEY=your-secret-key
NEXT_PUBLIC_MY_PUBLIC_KEY=your-public-key
```
